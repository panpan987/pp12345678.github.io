MinIO的安装与使用

发表于2021-03-20|更新于2021-03-20|运维
|字数总计:681|阅读时长:2分钟|阅读量:

MinIO简介

  • MinIO是高性能的对象存储服务器，它非常适合存储非结构化数据，如图片、视频、音频、日志文件等，单个文件大小可以从几KB到5TB。
  • MinIO服务器足够轻量，提供的图形化查看界面很不错。
  • 图标是一只仙鹤，官网地址：https://min.io/

安装MinIO

这里使用Docker安装MinIO

搜索镜像

使用命令：docker search minio

选第一个STARTS最多的准没错…

拉取镜像

使用命令：docker pull docker.io/minio/minio (说明：docker pull + 镜像的名字）

查看镜像

使用命令： docker images 这是查看所有的镜像信息，包括未运行的和已运行的

上面是刚刚下载的镜像，对上面红色标注做一下解释：

  • REPOSITORY：来自于哪个仓库；
  • TAG：镜像的标签，比如5.7、latest分别表示5.7版本、最新的版本；
  • IMAGE ID：镜像的ID，唯一的，如果两个ID完全相同，那么实际上它们指向同一镜像，只是标签名称不同；
  • CREATED：镜像最后的更新时间；
  • SIZE：镜像的大小，优秀的镜像一般体积都比较小；

运行镜像

运行镜像之后有了minio容器，镜像相当于类，容器相当于实例（对象）

说明：

  • -p 9000:9000 将宿主机（Linux系统）9000端口映射到minio容器 9000端口，通过访问宿主机的 ip:9000就访问到容器
  • -d --restart=always Docker重启后容器也会重启
  • -e "MINIO_ACCESS_KEY=自定义AK" -e "MINIO_SECRET_KEY=自定义SK" \自定义AK和SK，相当于用户名和密码，访问容器必须需要这两个KEY
  • -v /home/data:/data -v /home/config:/root/.minio 将宿主机上的数据目录、配置与容器内目录相映射

查看容器运行情况

使用命令 docker ps -a

注意查看STATUS，有Up…表示运行着，如果是Exit..表示未运行

查看安装logs信息

命令 docker logs 6189eedfc191 （docker logs + CONTAINER ID）

登录客户端

浏览器输入linux的ip+端口，如101.xx.xx.xx:9000

输入刚才自定义的AK和SK，进入图形化界面

创建桶

文件存储在桶内，桶可以认为是文件夹。minio桶名称(bucket name)命名不支持下划线，支持中划线。

点击右下角+号创建桶

之后就能轻松愉快地存储数据了，可以直接在浏览器中往桶内存数据，也可以通过MinIO SDK，如通过Java客户端API调用，参考：

https://docs.min.io/docs/java-client-api-reference.html
